BlendIO / blend2glb Manual

BlendIO 是一个面向 Blender 4.5.x 的 Rust 读取与导出工具集。 它可以在不依赖 Blender 本体的前提下，直接读取 little-endian .blend 文件，并提供诊断型 CLI、结构化库 API，以及 .blend -> .glb 的直接导出能力。

当前仓库内包含两个可执行入口： blendio 用于读取和诊断， blend2glb 用于导出 GLB。 默认命令行输出是逐行可读文本，只有显式指定 --json 或 --report-json 时才会输出 JSON。

Target: Blender 4.5.x Runtime: No Blender Required Formats: raw / gzip / zstd .blend Export: single-file .glb

项目概览

BlendIO 的底层读取路径不是硬编码结构体，而是依赖 DNA1 / SDNA 做动态结构解析。也就是说， 先把文件头和块表读出来，再从文件内部的 SDNA 构建字段布局， 然后在这个基础上做结构视图、指针解析、ID 汇总和更高层的摘要提取。

组件 作用
blendio 读取 .blend，查看块表、SDNA、ID、摘要信息。
blend2glb 从 .blend 直接导出单文件 .glb。
blendio crate 给别的 Rust 程序复用，直接读取文件、遍历块、做导出。

运行时依赖

打包好的 blendio.exe 和 blend2glb.exe 在目标机器上都不需要安装 Blender。只要可执行文件本身能运行，并且输入文件符合当前支持范围，就可以直接解析或导出。

仓库里的集成测试会调用本机 Blender 4.5 来生成样本文件，这只是测试与逆向验证用，不是运行时依赖。

支持范围

读取层

  • 支持未压缩、gzip、zstd 三种输入。
  • 支持 12 字节 legacy 头和 17 字节 large header。
  • 支持 BHead4、SmallBHead8、LargeBHead8。
  • 完整解析 DNA1 / SDNA 的 NAME / TYPE / TLEN / STRC。
  • 支持 old-pointer 索引、结构视图、指针解析、数组块读取、ListBase 遍历。
  • 当前完整支持路径为 Blender 4.5.x little-endian 文件。

info 摘要层

  • 文件头、压缩方式、块数量、ID 数量、SDNA 统计。
  • 块代码分布和 ID 代码分布。
  • Scene：相机、世界、帧范围、FPS、分辨率、渲染引擎、输出路径。
  • Object：类型、父子、data 目标、TRS、旋转模式、材质槽、动画、modifier、外部库来源。
  • Mesh：顶点/边/loop/poly 计数、legacy 指针存在性、UV 属性、顶点属性层。
  • Collection、Library、Image、Action、Text、Camera、Light、Material、World。

导出层

  • 首版导出 Mesh、Camera、Light、对象层级、对象 TRS 动画。
  • 支持 Principled BSDF 的 glTF 原生 PBR 子集。
  • 支持 packed 图片嵌入 GLB。
  • 支持 Triangulate、Mirror、Array 三类 modifier 的受限求值。
  • 对不支持特性走 warning / report，而不是静默吞掉。

构建与打包

开发模式运行读取器：

cargo run --bin blendio -- info "D:\test\demo.blend"

开发模式运行导出器：

cargo run --bin blend2glb -- "D:\test\demo.blend" "D:\test\demo.glb"

发布版构建：

cargo build --release

生成文件默认位于：

target/release/blendio.exe
target/release/blend2glb.exe

blendio CLI

默认输出是逐行可读文本，更适合直接在终端看结果。只有显式加 --json 时才会把 JSON 打到标准输出。

查看帮助

blendio --help

读取完整摘要

blendio info "D:\test\demo.blend"

查看块表

blendio blocks "D:\test\demo.blend"

查看全部 ID

blendio ids "D:\test\demo.blend"

查看 SDNA 概览

blendio sdna "D:\test\demo.blend"

查看某个结构体字段

blendio sdna "D:\test\demo.blend" --type Object

把 JSON 打到终端

blendio --json info "D:\test\demo.blend"

把 JSON 写到文件，但终端仍保持文本输出

blendio --json-out "D:\test\demo_info.json" info "D:\test\demo.blend"
如果你希望“终端是 JSON”并且“同时另存 JSON 文件”，可以把 --json 和 --json-out 一起用。

blend2glb CLI

blend2glb 默认会输出逐行导出报告，包含 mesh/material/animation 数量、 warning 数量、跳过对象数量和不支持特性数量。

基础用法

blend2glb "D:\test\demo.blend" "D:\test\demo.glb"

常用参数

参数 含义
--no-cameras 不导出 Camera。
--no-lights 不导出 Light。
--no-animation 不导出对象 TRS 动画。
--strict 把 warning 升级为失败。
--report-json <path> 把导出报告写到 JSON 文件。

示例

blend2glb `
  "D:\test\demo.blend" `
  "D:\test\demo.glb" `
  --report-json "D:\test\demo_export_report.json"

作为 Rust 库引用

如果你的另一个 Rust 项目想直接复用这个库，最简单的方式是先用 path dependency。

Cargo.toml

[dependencies]
blendio = { path = "../BlendIO_rustc" }

读取文件并拿高层摘要

use blendio::{BlendFile, summarize};

fn main() -> Result<(), Box<dyn std::error::Error>> {
    let file = BlendFile::open("D:\\test\\demo.blend")?;
    let summary = summarize(&file)?;

    println!("blend version: {}", summary.header.file_version);
    println!("object count: {}", summary.objects.len());

    for object in &summary.objects {
        println!("{} -> {}", object.name, object.object_type);
    }

    Ok(())
}

读取底层块表

use blendio::BlendFile;

fn main() -> Result<(), Box<dyn std::error::Error>> {
    let file = BlendFile::open("D:\\test\\demo.blend")?;

    for block in file.blocks() {
        println!(
            "{} len={} old_ptr=0x{:X}",
            block.code,
            block.len,
            block.old_ptr
        );
    }

    Ok(())
}

按 old-pointer 解析结构体

use blendio::BlendFile;

fn main() -> Result<(), Box<dyn std::error::Error>> {
    let file = BlendFile::open("D:\\test\\demo.blend")?;

    for id_block in file.ids() {
        let view = match id_block.struct_view() {
            Ok(view) => view,
            Err(_) => continue,
        };

        if id_block.header().code.as_string() == "OB" {
            if let Some(name) = view
                .field("id")
                .and_then(|v| v.as_struct_view())
                .and_then(|id| id.field("name"))
                .and_then(|v| v.as_c_string())
            {
                println!("object raw id name = {}", name);
            }
        }
    }

    Ok(())
}

直接导出 GLB

use blendio::{ExportOptions, export_glb};

fn main() -> Result<(), Box<dyn std::error::Error>> {
    let options = ExportOptions::default();
    let report = export_glb(
        "D:\\test\\demo.blend",
        "D:\\test\\demo.glb",
        &options,
    )?;

    println!("meshes: {}", report.exported_mesh_count);
    println!("warnings: {}", report.warnings.len());
    Ok(())
}

JSON 与文本输出策略

  • blendio 默认输出文本。
  • blendio --json ... 输出 JSON 到 stdout。
  • blendio --json-out file.json ... 把 JSON 另存到文件，同时 stdout 仍然保持文本。
  • blend2glb 默认输出文本报告。
  • blend2glb --report-json report.json ... 把导出报告另存为 JSON。

限制与注意事项

  • 读取层当前完整支持 Blender 4.5.x little-endian 文件。
  • big-endian 文件会明确返回不支持。
  • 不支持写回 .blend。
  • 不递归打开外部链接库 .blend。
  • GLB 导出首版不追求和 Blender 官方 glTF 导出逐字节一致。
  • 导出层暂不支持骨骼/蒙皮、shape key、几何节点完整求值、复杂约束求值、外部贴图完整复刻。
如果你后面要把它扩成资产检查器、批量转模工具、或嵌入自己的引擎工具链， 推荐优先走库 API，而不是解析 CLI 文本输出。
